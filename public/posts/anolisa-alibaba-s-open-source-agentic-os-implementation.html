ANOLISA: Alibaba's Open Source Agentic OS Implementation

March 30, 2026

在 Agentic OS 的探索浪潮中，阿里巴巴开源了 ANOLISA（Agentic Nexus Operating Layer & Interface System Architecture）——一个完整的 Agentic OS 实现栈。基于 Anolis OS 构建，ANOLISA 旨在为 AI Agent 工作负载提供最佳实践的服务端操作系统实现。本文将深入分析其架构设计、核心组件和技术创新。

项目概述

ANOLISA 是阿里巴巴 Anolis OS 的 Agentic 演进版本，代表了业界首个完整的 Agentic OS 开源实现。它不是简单的 AI 助手集成，而是从操作系统层面重新设计，为 AI Agent 提供原生的系统级支持。

核心定位： 为 AI Agent 工作负载构建的服务端操作系统，提供从安全、可观测性到技能编排的完整基础设施。

整体架构

ANOLISA 采用分层架构设计，由四大核心组件构成完整的 Agentic OS 栈：

核心组件深度解析

1. Copilot Shell — AI 驱动的终端助手

Copilot Shell 是 ANOLISA 的用户交互层，基于 Qwen Code v0.9.0 构建，提供自然语言编程能力。

核心特性：

  • 自然语言编程：用自然语言描述即可修改代码、实现功能或修复 Bug，无需手动编写命令
  • 代码分析与导航：理解整个项目结构，回答代码相关问题，支持代码库级别的语义理解
  • 多工具编排：在单次会话中集成文件、Shell、搜索、Web、LSP 和 MCP 工具
  • 交互式 Shell：/bash 命令进入交互式 Shell，输入 exit 返回，无缝切换
  • 技能系统：本地 + 远程技能发现，支持优先级回退（项目 > 用户 > 扩展 > 远程）
  • Hooks 系统：PreToolUse 事件，在工具调用前拦截处理，实现安全策略注入
  • Git 工作流自动化：自动化提交、分支创建、冲突解决和发布说明生成
  • 多模型支持：支持 Qwen OAuth、阿里云百炼、OpenAI 兼容 API、Anthropic 和 Google GenAI
  • PTY 模式：完整的伪终端支持，包括 sudo 命令

技术架构：

采用 npm workspaces 的 monorepo 布局：

  • packages/cli：终端 UI 层 — 输入处理、命令解析、Ink/React 渲染
  • packages/core：后端核心 — AI 模型通信、提示词构建、工具编排
  • packages/test-utils：共享测试工具

2. Agent Sec Core — OS 级安全内核

Agent Sec Core 是面向 AI Agent 的 OS 级安全内核，提供系统加固、资产完整性校验与安全决策的完整防护链。这是 ANOLISA 最具创新性的组件之一。

核心原则：

  1. 最小权限：Agent 仅获得完成任务所需的最小系统权限
  2. 显式授权：敏感操作必须经过用户明确确认，禁止静默提权
  3. 零信任：Skill 间互不信任，每次操作独立鉴权
  4. 纵深防御：系统加固 → 资产校验 → 安全决策，任一层失守不影响其他层
  5. 安全优先于执行：当安全与功能冲突时，安全优先；存疑时按高风险处理

三层安全防护架构：

┌─────────────────────────────────────────────┐
│              Agent Application              │
├─────────────────────────────────────────────┤
│         安全决策流程（风险分级与处置）          │
├─────────────────────────────────────────────┤
│  Phase 3: 最终安全确认                       │
├─────────────────────────────────────────────┤
│  Phase 2: 关键资产保护 (GPG + SHA-256)       │
├─────────────────────────────────────────────┤
│  Phase 1: 系统安全加固 (loongshield)         │
├─────────────────────────────────────────────┤
│              Linux Kernel                   │
└─────────────────────────────────────────────┘

风险分级与处置：

风险等级 典型场景 处置策略
低 文件读取、信息查询、文本处理 允许，沙箱内执行
中 代码执行、包安装、调用外部 API 沙箱隔离 + 用户确认
高 读取 .env/SSH 密钥、数据外发、修改系统配置 阻断，除非用户显式批准
危急 Prompt injection、secret 外泄、禁用安全策略 立即阻断 + 审计日志 + 通知用户

沙箱策略模板：

linux-sandbox 提供 3 种内置策略模板（基于 Rust + bubblewrap + seccomp）：

模板 文件系统 网络 使用场景
read-only 全盘只读 禁止 只读操作：ls、cat、grep、git status 等
workspace-write cwd + /tmp 可写，其余只读 禁止 构建、编辑、脚本执行等需要写文件的操作
danger-full-access 无限制 允许 ⚠ 保留模板，仅供特殊场景手动指定

3. AgentSight — eBPF 可观测性工具

AgentSight 是基于 eBPF 的 AI Agent 可观测性工具，提供零侵入式的 LLM API 调用监控、Token 用量统计、进程行为追踪和 SSL/TLS 流量捕获。

核心特性：

  • 零侵入式监控：通过 eBPF 内核探针捕获事件，无需修改 Agent 代码或配置
  • SSL/TLS 流量解密：基于 uprobe 拦截 OpenSSL/GnuTLS 库调用，捕获加密连接的明文 HTTP 流量
  • LLM Token 精确计量：集成 Hugging Face tokenizer，支持 Qwen 等系列模型的精确 Token 计数
  • AI Agent 自动发现：扫描 /proc 并监控 execve 事件，动态检测系统上运行的 AI Agent 进程
  • 流式响应支持：解析 Server-Sent Events (SSE)，追踪 LLM 流式响应
  • 审计日志：完整记录 LLM 调用和进程操作的结构化审计轨迹
  • 云端集成：原生支持导出至阿里云 SLS（日志服务）进行集中化日志分析

统一数据流水线架构：

┌──────────┐    ┌────────┐    ┌────────────┐    ┌──────────┐    ┌───────┐    ┌─────────┐
│  Probes  │───▶│ Parser │───▶│ Aggregator │───▶│ Analyzer │───▶│ GenAI │───▶│ Storage │
└──────────┘    └────────┘    └────────────┘    └──────────┘    └───────┘    └─────────┘
  eBPF 事件      HTTP/SSE      请求-响应          Token/审计      语义         SQLite /
  (内核态)       结构化提取     关联聚合           信息提取        事件构建     SLS 导出

eBPF 探针：

探针 源文件 说明
sslsniff src/bpf/sslsniff.bpf.c 通过 uprobe 挂钩 SSL_read/SSL_write，捕获加密连接的明文数据
proctrace src/bpf/proctrace.bpf.c 追踪 execve 系统调用，捕获命令行参数，构建进程树
procmon src/bpf/procmon.bpf.c 轻量级进程监控，追踪进程创建/退出事件（用于 Agent 发现）

支持的 LLM 提供商：

  • OpenAI / OpenAI 兼容 API
  • Anthropic（Claude，包括缓存 Token 处理）
  • Google Gemini
  • 通义千问 Qwen（支持原生 Chat Template）

4. OS Skills — 运维技能库

OS Skills 是精心策划的技能库，涵盖系统管理、监控、安全、DevOps 和云集成等场景。每个技能都是可复用的、经过验证的最佳实践。

技能分类：

  • System Admin：系统管理技能（内核升级、存储调整、Shell 脚本、正则表达式、备份恢复、Anolis 管理）
  • Security：安全相关技能（待扩展）
  • DevOps：DevOps 相关技能（待扩展）
  • Cloud Integration：云集成技能（待扩展）

技能结构：

每个技能目录包含：

  • SKILL.md：技能描述和使用指南
  • scripts/：技能脚本
  • references/：参考文档和示例
  • examples.md：使用示例

技术创新与亮点

1. 完整的 Agentic OS 栈

ANOLISA 不是简单的 AI 助手集成，而是从操作系统层面重新设计，提供了从用户交互、技能编排、安全防护到可观测性的完整基础设施。这是业界首个完整的 Agentic OS 开源实现。

2. 三层纵深安全架构

Agent Sec Core 的三层安全架构（系统加固 → 资产校验 → 安全决策）为 AI Agent 提供了企业级的安全保障。特别是基于 Rust 的沙箱执行器和 GPG 签名校验机制，确保了 Agent 在受控、可审计、最小权限的环境中运行。

3. 零侵入式可观测性

AgentSight 基于 eBPF 的零侵入式监控技术，无需修改 Agent 代码或配置即可捕获 LLM API 调用、Token 消耗和进程行为。SSL/TLS 流量解密能力更是突破了传统监控的盲区。

4. 自然语言编程

Copilot Shell 提供的自然语言编程能力，让用户可以用自然语言描述任务，Agent 自动转换为具体的系统操作。这降低了使用门槛，提高了开发效率。

5. 多语言协同

ANOLISA 采用多语言协同架构：

  • Node.js：Copilot Shell（终端 UI 和 AI 模型通信）
  • Rust：AgentSight（eBPF 探针）和 linux-sandbox（沙箱执行器）
  • Python：OS Skills（技能库）和 Agent Sec Core（安全策略）

这种多语言架构充分发挥了各语言的优势，实现了性能、开发效率和生态的最佳平衡。

应用场景

1. AI 辅助系统运维

运维人员可以通过 Copilot Shell 用自然语言描述运维任务，Agent 自动执行系统检查、故障诊断、性能优化等操作，大幅提高运维效率。

2. 自动化 DevOps 流程

通过 OS Skills 库，Agent 可以自动执行 CI/CD 流程、容器编排、配置管理等 DevOps 任务，减少人工干预。

3. 安全审计与合规

Agent Sec Core 的安全防护机制和 AgentSight 的审计日志功能，为企业提供了完整的安全审计和合规检查能力。

4. 成本优化

AgentSight 的 Token 精确计量功能，帮助企业监控和优化 LLM API 调用成本，实现精细化的成本管理。

总结

ANOLISA 代表了 Agentic OS 领域的重要里程碑。作为业界首个完整的 Agentic OS 开源实现，它不仅提供了从用户交互到系统底层的完整基础设施，更展示了如何将 AI Agent 能力深度集成到操作系统中。

其三层纵深安全架构、零侵入式可观测性、自然语言编程能力和多语言协同设计，为 Agentic OS 的发展提供了宝贵的实践经验和技术参考。对于企业级 AI Agent 应用，ANOLISA 提供了一个可信赖、可扩展、可观测的生产级解决方案。
