Aria：扔掉所有人手工策略，让 Claude Code 自由写 Coq 证明——Iris 分离逻辑 4,257 引理零失败

July 13, 2026

扔掉所有人的手工策略，让 Claude Code 自由写 Coq 证明——Aria 证明了 Iris 分离逻辑的全部 4,257 个引理，零失败，零人类干预

形式化验证有最强的正确性保证——但它的代价是"专家 proof engineer 一个 tactic 一个 tactic 地手写证明"。过去十年，ML 研究一直在试图自动化这个过程，但有一个共同假设：必须把模型塞进人类设计的证明策略中——tactic-by-tactic tree search、premise retrieval、divide-and-conquer 分解。这些系统的最好成绩是证明 12%–48% 的目标定理——而且这些 benchmark 刻意回避了最难的问题（并发、分离逻辑、共享内存）。

这篇 Anonymous（2026 年 7 月）的论文——Harnessing Code Agents for Automatic Software Verification——提出了一个激进的对立面：扔掉所有手工策略，让一个通用的 code agent（Claude Code）自由决定每一步怎么走，外面只套一个不会说谎的 verification harness。结果是：在形式化验证中最难的基准——Iris 分离逻辑——上证明了每一个目标引理（4,257 个），零失败，零人类专家干预。

核心设计：Strategy lives in the agent, not in the system

Aria 的三层架构代表了这篇论文的方法论贡献：

Model Layer：可互换的 LLM backend。本文使用 Claude Opus 4.7。
Agent Layer：所有的战略决策在这里发生。agent 决定用哪个引理、找哪些相似证明、如何分解子目标、何时回溯。
Harness Layer：系统的信任锚点。Coq kernel 验证 + per-tactic timeout + 禁止 Admitted/admit + 引理覆盖检查 + Iris linter + shell 安全检查。

论证的精髓在于：

所有这些战略决策——用哪个引理、如何分解目标、何时放弃——应该由 agent 自己根据 context 逐引理地做，而非被一个统一的、人类设计的策略定死。Agent 先找相似引理，模仿其证明模式，而非从零构造——这是惊人的人类化的证明方式。

HHL：声明式 Harness 语言

Aria 被低估的贡献是 HHL（Harness Hook Language）——一种声明式语言，把 verification harness 从 ad-hoc 胶水代码变成了 first-class, auditable artifact：

hook no_banned: Write|Edit(file, original) = no_banned_fn
hook safe_command: Bash(cmd) = safe_command_fn
hook verify_proof: Turn(files) = verify_proof_fn

workflow prove_lemmas(lemmas) {
  for lemma in lemmas {
    while !proved && tries < 30 {
      let r = invoke(project, prompt ++ feedback,
                     pre:[no_banned, safe_command],
                     post:[verify_proof])
      feedback = "Previous error: " ++ r.result
    }
  }
}

pre-hooks 在 agent 动作前运行（可 Deny），post-hooks 在动作后运行（可 Ok/Err）。HHL 编译到 Claude Code SDK（PreToolUse/PostToolUse hooks），可重定向到其他 agent runtime。这是一个让 harness = spec 的语言设计。

错误的精确反馈是秘密武器

Aria 的 harness 不只说"证明失败"——它返回三个东西：

  1. 失败的步骤（第几行）
  2. Coq 的错误消息（如 Unable to unify "P" with "Q"）
  3. 当前待证明的目标状态（pending goal）

这本质上是 guided search against ground truth。agent 提出候选 → harness 精准指出"哪里错了 + 还需证明什么" → agent 修订 → 循环。300 秒 per-tactic timeout 防止发散。最多 30 次 retry，每次恢复同一 session 携带完整错误历史。

多 agent 协作循环进一步分解了任务：Extractor（发现未证明引理）→ Prover（写第一版证明）→ Fixer（收到精确错误后修订）→ Polish（风格润色，如压缩 bare done 序列为 try done）。

在 CoqStoq 中 之前 LLM prover 表现最差的 benchmark（Rango 仅 12.5%）——Aria 证明了全部 318 个引理，75.5% first-try，平均 0.38 retries。有一个 outlier：nfa_ofP（双向有限自动机构造的正确性）花了 5.8 小时，比最难的 Iris 引理还长。

iris-lean：72 lemmas / 0 failures（Lean 4 端口）

在 Lean 4（而非 Coq）上，针对 从未存在过证明 的引理——Aria 证明了全部 72 个，90.3% first-try，不到两小时完成。这排除了"模型只是记住了训练数据中的证明"的替代解释。

私有 vs 开源模型的诚实对比

在 40 个较短引理上，开源模型（Kimi K2.6）也能证明全部——72.5% vs 55% first-try，169s vs 1253s/lemma。但在 长证明（>50 行 Coq）上暴露了真正的 capability gap：开源模型即使耗尽 30 次 retry 也经常失败，Opus 4.7 可靠地构造和保持 200+ 行的证明。

"零失败"不是 benchmark score——它是系统性质

这是这篇论文最被低估的一点。当你之前的系统证明"40.4% 的定理"，这意味着你还需要人类处理剩下的 59.6%。当 Aria 证明"100% 的定理，零失败"——这意味着 人类不需要碰任何一个引理。从工程角度，这是"辅助工具"和"自主系统"的分界线。

"Trust is free" 是这个范式成立的关键：Coq kernel 拒绝任何不正确的证明。LLM 在其他领域臭名昭著的"自信但错误"在这里无法通过——难度从来不是"是否相信这个证明"，而是模型能否找到一个证明。

对 Harness Engineering 的启示

1. "Strategy in agent, verification in harness" 是可推广的范式

这不仅适用于 Coq 证明。任何有 ground-truth oracle 的 agent 任务——formal verification、unit test-based code generation、mathematical proof——都可以用"agent 自由探索 + harness 硬验证"替代精心设计的手工策略。Aria 的 4,257/4,257 vs. 之前最好的 48%，证明了"让 agent 自己决定策略 > 人类设计的固定策略"不只是一个哲学立场——它是实验事实。

2. HHL 是 harness 作为 first-class language 的参考实现

HHL 的 pre/post hooks + workspace + workflow 设计模式，让它成为 harness specification 的原型。这与 Lilian Weng 的"harness 应该像 OS 一样封装复杂性"形成完美的工程对偶——HHL 是实现这个理念的具体语言设计。

3. 精确错误反馈是反馈循环质量的瓶颈

Aria 的 harness 返回的不是"失败了"而是"第 4 行 apply HP 失败了，错误是无法统一 P 和 Q，pending goal 是 Q"。这种精确度让 agent 的修订有效。在 meta-agent-poc 的验证节点中，错误的粒度直接决定修正的效率——"哪个 spec 子句违规"比"spec 检查失败"有用得多。

4. "零人类干预的完整 campaign"定义了自主系统的上界

Aria 的运行时性质——380 小时，从启动到最后一个完成的证明，没有人检查目标、写 tactic、或手动重启——展示了当 harness 足够强时，agent 系统可以达到的自主性上限。这不是科幻：它已经在 Coq 和 Lean 上发生了。

局限与诚实标注

  • 模型依赖：主要结果基于 Claude Opus 4.7。开源模型在长证明上不足。如果模型不可用，结果可能无法复现。论文承认。
  • 仅 Coq + Lean：未在其他 ITP（Isabelle、Metamath）上验证。
  • Harness 本身需要人类设计：HHL 程序、hooks、rubric criteria 都是专家写的。paradox：你需要足够理解 Iris 的人来写 harness，但一旦写好，那个人就不需要碰证明了。这是我的推断。
  • 计算成本不低：Opus 4.7 × 380h ≈ 数万美元 API 开销。但论文论证这是合理的——雇 Coq 专家手工证明 4,257 个 Iris 引理要贵得多，而且实际没有人能做到。论文的论证。
  • 未在其他分离逻辑框架上测试：对 VST（C）、CakeML 的 proof infrastructure 的可迁移性未知。

一句话

之前的所有 LLM proof 系统都在问"如何让模型在人类设计的证明过程中更好地预测下一步"。Aria 问的是"如果让模型自己决定每一步，人类只负责验证它会怎样"——答案是全部 4,257 个引理，零失败，零人类干预。策略应该在 agent 的 context 中，而非系统的硬编码中。
